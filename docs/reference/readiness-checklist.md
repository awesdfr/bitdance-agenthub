# V1 Readiness Checklist

All items must be explicitly reviewed before v1 development starts.

| Item | Required evidence |
| --- | --- |
| 技术选型 | Runtime, database, UI, automation, and packaging choices are documented. |
| 核心类型定义 | Core Agent, workflow, memory, tool, artifact, and permission types are present. |
| 安全基础 | Secrets, permissions, approvals, sandbox, audit, and abuse boundaries exist. |
| Phase 0范围 | First build scope is explicit and unsafe/deferred integrations are excluded. |
| 开发环境 | Required local tools and commands are known. |
| 团队 | Product, engineering, design, security, docs, and support roles are acknowledged. |
| 文档 | Documentation architecture, glossary, FAQ, troubleshooting, and reference cards exist. |
| 法律 | License, forbidden uses, data policy, and release obligations are reviewed. |
